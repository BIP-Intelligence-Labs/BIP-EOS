# M-006.2 --- Command Dispatcher

**Milestone:** M-006.2\
**Status:** In Progress\
**Release:** Atlas\
**Version:** 0.1.0 Genesis

------------------------------------------------------------------------

# Objective

Introduce the UEOS Enterprise Command Dispatcher.

This milestone transforms UEOS from a bootable runtime into an
interactive engineering operating system capable of dispatching commands
through a unified command architecture.

------------------------------------------------------------------------

# Scope

The dispatcher is responsible for:

-   Registering commands
-   Alias resolution
-   Dispatching commands
-   Unknown command handling
-   Command discovery
-   Interactive shell integration

The dispatcher does **not** implement business logic. Business logic
belongs to individual command implementations.

------------------------------------------------------------------------

# Design Principles

## Single Responsibility

The dispatcher routes commands only. Commands execute business logic.

## Open / Closed Principle

New commands are added by registration:

``` python
dispatcher.register(GraphCommand())
```

No dispatcher modification is required.

# Responsibilities

## Dispatcher

-   Register commands
-   Resolve aliases
-   Parse command line
-   Dispatch execution
-   Return exit codes

## Command

Each command shall:

-   Validate input
-   Execute business logic
-   Return an exit code
-   Report failures

## Shell

Responsible for:

-   User input
-   Prompt
-   Keyboard interrupts
-   Session lifecycle

# Atlas Engineering Principle

Every capability added to UEOS shall be exposed through the Command
Dispatcher.

No subsystem shall bypass the dispatcher.

The dispatcher is the official execution gateway into UEOS.
